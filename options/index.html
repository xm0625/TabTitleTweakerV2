    Please take some time to read the Instructions and Examples!

  • Syntax (one rule per line):

    operation, pattern, text
    operation, pattern, text

    Operations

    • rename : completely replace the title with text
    • prefix : add text to the beginning of the page title
    • suffix : add text to the end of the page title
    • revert : reset the title to its original value (for overriding other rules; please see examples)

    Tips

    • Titles may not be altered until the page has finished loading.
    • The TTT icon will appear in the address bar if Tab Title Tweaker has found matching rules for the page.
    • Use optional * wildcards in the pattern part of the rule to match one or more characters (anything but spaces).
      See all examples.
    • If you omit http:// or https:// then both will match automatically.
      See example #4 to see how to match only one protocol.
    • mysite.com/mydir is not the same as mysite.com/mydir*
      See example #3.
    • Rules are actioned in order, so if multiple rules match a page, the lower rule will override the higher one.
      See example #3.
    • Because revert rules make sure tab title stays the same (overriding previous rule matches), you probably want to put these rules at the end of the list.
    • Some sites have javascript-updated page titles (AJAX-powered sites like Gmail, where the tab title may update to show the number of unread emails in your inbox). TTT will change these page titles if rules match, but the customized titles may be replaced by the site's underlying javascript. I may be able to overcome this in future version of TTT.

  • Example 1: match subdirectories unless overridden

    Rule: prefix, *.devserver.com, DEV
    • mysite.devserver.com/mypage
      The tab title changes to "DEV MyPage" (original page title was "MyPage")

    Example 2: wildcard precision

    Rule: rename, *.devserver.com/xyz/*, Foo
    • mysite.devserver.com/xyz
      The tab title won't change
    • mysite.devserver.com/xyz/
      The tab title won't change
    • mysite.devserver.com/xyz/mypage
      The tab title changes to "Foo"

    Example 3: matching a single page + overriding rules

    Rule: suffix, devserver.com/xyz/page-x, Foo
    Rule: revert, devserver.com/xyz/page-x*
    • devserver.com/xyz/page-x
      The tab title changes to "Page X Foo"
    • devserver.com/xyz/page-xyz
      This matches the second rule in the example. Because it's a revert rule, the tab title will won't change.

    Example 4: matching only http

    Rule: rename, http://mysite.com/xyz/page-x, Foo
    • http://mysite.com/xyz/page-x
      The tab title changes to "Foo"
    • https://mysite.com/xyz/page-x
      The tab title won't change
    • Changing the rule to include the "s" will give you the opposite effect
